## 功能特性

- **全面格式检查** - 覆盖规范第3.2~3.7节、第4.1~4.9节所有可自动检查项
- **智能格式修复** - 自动修复字体、字号、段落、引用标注、页眉页脚等格式问题
- **页码分节管理** - 插入分节符，设置正确的页码格式（目录罗马数字 + 正文阿拉伯数字）
- **示例文档生成** - 从零生成符合规范的示例docx文档
- **结构化报告** - 输出详细的检查报告，包含错误、警告和通过项

## 安全特性

**所有脚本均创建副本操作，不会修改原文件：**

| 脚本 | 输入文件 | 输出文件 |
|------|----------|----------|
| `check_thesis.py` | 论文.docx | 只读检查，无输出文件 |
| `fix_thesis.py` | 论文.docx | 论文_fixed.docx |
| `fix_pagenum.py` | 论文.docx | 论文_页码修复.docx |
| `generate_thesis.py` | 无 | 创建新文件 |
| `verify_thesis.py` | 论文.docx | 只读验证，无输出文件 |

## 快速开始

### 安装依赖

```bash
pip install python-docx
```

### 使用方法

#### 1. 检查格式
```bash
python scripts/check_thesis.py 论文.docx
```

输出示例：
```
============================================================
  湖南科技大学毕业论文格式检查
  文件: 论文.docx
============================================================
  ✓ 通过: 227 项
  ✗ 错误: 0 项
  ⚠ 警告: 0 项
============================================================
```

#### 2. 修复格式
```bash
python scripts/fix_thesis.py 论文.docx [输出文件.docx]
```

#### 3. 修复页码（分节符）
```bash
python scripts/fix_pagenum.py 论文.docx
```

#### 4. 生成示例文档
```bash
python scripts/generate_thesis.py 输出文件.docx
```

#### 5. 验证格式
```bash
python scripts/verify_thesis.py 论文.docx
```

## 规范速查

### 页面设置
- 纸张：A4 (210mm×297mm)
- 页边距：上30mm、下25mm、左25mm、右25mm
- 页眉距离：20mm
- 页脚距离：15mm

### 页眉页脚
- 页眉：湖南科技大学本科生毕业设计（论文），五号宋体居中
- 页码：前置部分 -i- 罗马数字，正文 -1- 阿拉伯数字，五号Times New Roman居中

### 标题格式
- 章标题：宋体加粗小二号(18pt)居中，段前段后各0.5行
- 节标题：宋体加粗四号(14pt)顶格，段前段后各0.5行
- 条标题：宋体加粗小四号(12pt)顶格，段前段后各0.5行

### 正文格式
- 宋体小四号(12pt)
- 段前段后不空行
- 1.25倍行距
- 首行缩进2字符(24pt)

### 标题规范
- 章标题≤15字，无标点符号
- 论文题目≤20字，不设副标题
- 章编号使用大写数字（如"第三章"而非"第3章"）

### 关键词
- 数量：3~5个
- 中文：关键词标签用黑体，词条用宋体
- 英文：Keywords标签加粗Times New Roman，词条Times New Roman

### 参考文献
- 标题：宋体四号加粗居中
- 条目：顶格，五号宋体，英文部分Times New Roman
- 标点：半角

### 表格插图
- 三线表，表序按章编排(表X.Y)
- 表名居中无标点，置于表上
- 图序按章编排(图X.Y)，图题在图下
- 表内文字：中文五号宋体，英文五号Times New Roman

### 另起页
- 中英文摘要、目录、每章、参考文献、致谢、附录需另起页

## 常见问题

### Q: 页码格式不对怎么办？
A: 先运行 `fix_pagenum.py` 插入分节符，再运行 `fix_thesis.py` 修复页脚字体。

### Q: 引用标注不是上标？
A: 运行 `fix_thesis.py` 自动将 [1], [2,3] 等设为上标。注意：以（1）、（2）等开头的款/项标题不会被误处理。

### Q: 字体显示不正确？
A: `fix_thesis.py` 同时设置中文字体和东亚字体，确保在Word中正确显示。

### Q: 文件被锁定？
A: 关闭Word后再运行脚本，或保存到不同文件名。

### Q: 检查报告中"页码应居中"但实际已居中？
A: 这是已知问题，页码居中检测可能误报。请手动检查页脚设置，如果确实已居中可忽略该警告。

### Q: 公式检查显示"公式应居中"但实际已居中？
A: 公式检测依赖于MathType或OMML对象识别，可能不准确。请手动确认公式格式。

### Q: 文档结构检查报告缺少摘要/引言/结论？
A: 如果文档确实缺少这些章节，需要手动添加。如果是检查误判，请确认章节标题是否符合规范格式（如"第一章 引言"）。

### Q: 参考文献条目字体检查不通过？
A: 规范要求参考文献中中文使用宋体，英文使用Times New Roman。运行 `fix_thesis.py` 会自动修复。

### Q: 如何检查关键词数量？
A: `check_thesis.py` 会自动检查关键词数量（3-5个），并在检查报告中显示。

### Q: 表格检查显示错误？
A: 对于复杂表格（合并单元格、不规则结构），检测可能不准确。请手动检查表格格式是否符合三线表规范。
